# binary-mlp-from-scratch

Кастомная реализация двуслойного перцептрона (MLP) с полным циклом обработки данных, статистическим EDA и сравнением с эталонными библиотеками. Проект выполнен в рамках модуля ML8 Школы 21.

---

## О проекте

Проект представляет собой **исследование** задачи бинарной классификации на датасете [Don't Get Kicked](https://www.kaggle.com/c/DontGetKicked) с Kaggle. Основная цель — реализовать многослойный перцептрон **с нуля** на чистом NumPy, продемонстрировав понимание:

- прямого и обратного распространения ошибки (backpropagation);
- работы оптимизаторов (SGD, Adam) и L2-регуляризации;
- построения пайплайна обработки данных;
- системного подхода к отбору признаков на основе статистических тестов и оценки важности.

Ключевая особенность проекта — **комплексный EDA**, который позволил обоснованно удалить неинформативные и нестабильные признаки, снизив размерность и улучшив обобщающую способность модели.

## Описание модулей проекта

### `src/MLP.py`
**Класс `MLP`** – основная реализация двуслойного перцептрона (многослойного перцептрона с одним скрытым слоем) для бинарной классификации.  
**Возможности:**
- Поддерживаемые активации: `sigmoid`, `relu`, `cos`, `tanh`, `elu`.
- Оптимизаторы: `sgd` (стохастический градиентный спуск) и `adam` (реализован вручную).
- L2-регуляризация (коэффициент `l2_lambda`).
- Инициализация весов по Xavier (для сигмоида, тангенса) или He (для ReLU, ELU, косинуса).
- Ранняя остановка с восстановлением лучших весов по валидационному Gini.

**Публичные методы:**
- `fit(X, y, X_val, y_val, ...)` – обучение модели с поддержкой валидации, ранней остановки и логирования.
- `predict_proba(X)` – возвращает вероятности для классов 0 и 1 (массив `(n_samples, 2)`).
- `predict(X, threshold=0.5)` – бинарные предсказания.

Код написан с акцентом на модульность: прямой проход, вычисление градиентов, обновление параметров и генерация батчей вынесены в отдельные приватные методы.

---

### `src/data_pipeline.py`
**Функция `prepare_data()`** – выполняет полный пайплайн предобработки данных от загрузки CSV до готовых массивов для обучения.

**Этапы:**
1. Загрузка данных, удаление идентификатора `RefId`, заполнение пропусков в категориях строкой `"Unknown"`.
2. Извлечение временных признаков из `PurchDate` (год, месяц, день недели, циклические `sin`/`cos`, количество дней с начала отсчёта).
3. Очистка поля `Model` от слов, присутствующих в `SubModel` и `Trim`.
4. Временной сплит (train:val:test = 33:33:33) с сохранением хронологии.
5. Удаление колонок, переданных в параметре `drop_cols` (по умолчанию – финальный список из EDA).
6. Разделение на целевую переменную (`IsBadBuy`) и признаки.
7. Приведение колонок `BYRNO` и `VNZIP1` к типу `object` для корректного кодирования.
8. Автоматическое определение категориальных и числовых колонок.
9. Кодирование категорий: OHE для низкокардинальных (≤30) с `drop='first'`, `CountEncoder` для высококардинальных с группировкой редких значений (`min_group_size`).
10. Импутация числовых пропусков медианой через `SimpleImputer`.
11. Масштабирование всех признаков через `QuantileTransformer` для приведения распределений к нормальному и устойчивости к выбросам.
12. Возврат массивов `X_train, X_val, X_test, y_train, y_val, y_test` в виде `np.ndarray`.

**Параметры:**
- `filepath` (str) – путь к CSV.
- `threshold` (int) – порог для выбора между OHE и CountEncoder.
- `min_group_size` (int) – минимальный размер группы для `CountEncoder`.
- `drop_cols` (list) – список колонок для удаления; если не передан, используется стандартный список.

---

### `src/data_processing.py`
Вспомогательный модуль с набором функций для первичной обработки данных, используемых внутри `data_pipeline.prepare_data()`.

**Функции:**
- `add_datetime_features(df, date_col)` – добавляет из даты колонки: `year`, `month`, `dayofweek`, `is_weekend`, и позже используются `month_sin/cos` и `days_since_start` в `preprocess_split`.
- `clean_model_from_sub_and_trim(row)` – очищает значение `Model` от слов, встречающихся в `SubModel` и `Trim`. Возвращает очищенную строку.
- `preprocess_split(df, base_date)` – добавляет `days_since_start` (относительно `base_date`) и циклические признаки месяца (`month_sin`, `month_cos`).
- `train_val_test_data_split(data)` – разбивает DataFrame на три части по времени (33/33/33) с сортировкой по `PurchDate`.

Эти функции являются чистыми трансформациями и не используют статистики целевой переменной, что гарантирует отсутствие утечек данных.

## Установка и запуск

### Требования

- Python 3.8+
- Установленные зависимости:

```bash
pip install numpy pandas scikit-learn category_encoders optuna tensorflow jupyter
```

### Клонирование и подготовка данных

```bash
git clone https://github.com/VadimGushchin/binary-mlp-from-scratch.git
cd binary-mlp-from-scratch
```

### Запуск ноутбука

```bash
jupyter notebook src/ML8.ipynb
```

### Использование в коде

```python
from src.MLP import MLP
from src.data_pipeline import prepare_data

# Загрузка и обработка данных
X_train, X_val, X_test, y_train, y_val, y_test = prepare_data(
    filepath="./src/data/training.csv",
    threshold=30,
    min_group_size=10
)

# Обучение модели
model = MLP(n_hidden=100, activation_func="sigmoid", optimizer="adam", l2_lambda=0.001)
model.fit(X_train, y_train, X_val, y_val, epochs=100, batch_size=32, verbose=True)

# Предсказание и оценка
proba = model.predict_proba(X_val)[:, 1]
gini = 2 * roc_auc_score(y_val, proba) - 1
print(f"Validation Gini: {gini:.4f}")
```

## Эксперименты и результаты

### Сравнение моделей

| Модель | Val Gini | Test Gini |
|--------|----------|-----------|
| Custom MLP (sigmoid) | **0.4367** | **0.4486** |
| sklearn MLPClassifier | 0.4303 | — |
| Keras/TensorFlow | 0.4356 | — |

### Выводы

- **Кастомная модель** показывает стабильное качество, сопоставимое с библиотечными реализациями, и даже немного превосходит их на валидации.
- **Тестовый Gini (0.4486)** не уступает валидационному, что свидетельствует об успешном снижении временного дрейфа и переобучения за счёт:
  - удаления нестабильных и низковажных признаков;
  - применения L2-регуляризации и ранней остановки;
  - использования устойчивого масштабирования (`QuantileTransformer`).
- Дополнительные эксперименты с активациями (`relu`, `cos`, `tanh`, `elu`) показали близкие результаты (Val Gini ≈ 0.437), что подтверждает робастность модели к выбору активации при правильной настройке.
